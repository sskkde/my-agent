# PostgreSQL Deployment Guide

This guide covers deploying the Agent Platform with PostgreSQL as the database backend.

## Prerequisites

- **PostgreSQL 15+** (recommended) or PostgreSQL 12+
- **Node.js 20+**
- **npm**

## Environment Variables

Set the following environment variables to enable PostgreSQL:

```bash
# Required: PostgreSQL connection string
DATABASE_URL=postgresql://user:password@host:5432/database

# Optional: Connection pool settings
PG_MAX_CONNECTIONS=10
PG_IDLE_TIMEOUT_MS=30000
PG_CONNECTION_TIMEOUT_MS=5000
```

### Connection String Format

```
postgresql://[user[:password]@][host][:port][/database][?param1=val1&param2=val2]
```

Example:
```bash
DATABASE_URL=postgresql://agent_user:secret@db.example.com:5432/agent_platform?sslmode=require
```

## Migration Commands

### Running Migrations Programmatically

```typescript
import { PostgresConnectionManager } from './src/storage/adapters/postgres/postgres-connection.js';
import { createPgMigrationRunner } from './src/storage/adapters/postgres/migration-runner.js';
import { pgStoreMigrations } from './src/storage/adapters/postgres/pg-migrations.js';

const connection = new PostgresConnectionManager({
  connectionString: process.env.DATABASE_URL,
});

await connection.open();

const runner = createPgMigrationRunner(connection);
await runner.init();
await runner.apply(pgStoreMigrations);

console.log('Migrations applied successfully');
await connection.close();
```

### Migration SQL Files

Individual migration files are available at:
```
src/storage/adapters/postgres/migrations/
├── 001_create_transcripts_table.sql
├── 002_create_sessions_table.sql
├── 003_create_users_and_auth_tokens.sql
├── ... (15 total SQL files)

```

To run migrations manually with `psql`:
```bash
psql $DATABASE_URL -f src/storage/adapters/postgres/migrations/001_create_transcripts_table.sql
```

## Connection Pooling Configuration

The PostgreSQL adapter uses `pg.Pool` for connection pooling. Configure these settings:

| Setting | Default | Description |
|---------|---------|-------------|
| `max` | 10 | Maximum connections in pool |
| `idleTimeoutMillis` | 30000 | Time before idle connections are closed |
| `connectionTimeoutMillis` | 5000 | Time to wait for connection acquisition |

### Production Recommendations

```bash
# For production with moderate load
PG_MAX_CONNECTIONS=20
PG_IDLE_TIMEOUT_MS=60000
PG_CONNECTION_TIMEOUT_MS=10000
```

## Schema Differences from SQLite

When migrating from SQLite to PostgreSQL, note these schema differences:

| SQLite | PostgreSQL |
|--------|------------|
| `INTEGER NOT NULL DEFAULT 1` | `BOOLEAN NOT NULL DEFAULT TRUE` |
| `INTEGER NOT NULL DEFAULT 0` | `BOOLEAN NOT NULL DEFAULT FALSE` |
| `TEXT` (JSON data) | `JSONB` |
| `datetime('now')` | `NOW()` |
| `AUTOINCREMENT` | `GENERATED BY DEFAULT AS IDENTITY` |
| `json_extract(col, '$.path')` | `col->>'path'` |

## Backup and Restore

### Backup

```bash
# Full database backup
pg_dump $DATABASE_URL > backup_$(date +%Y%m%d).sql

# Compressed backup
pg_dump $DATABASE_URL | gzip > backup_$(date +%Y%m%d).sql.gz

# Schema-only backup
pg_dump --schema-only $DATABASE_URL > schema_backup.sql
```

### Restore

```bash
# Restore from backup
psql $DATABASE_URL < backup_20260521.sql

# Restore compressed backup
gunzip -c backup_20260521.sql.gz | psql $DATABASE_URL
```

### Automated Backups

Set up a cron job for daily backups:
```bash
0 2 * * * pg_dump $DATABASE_URL | gzip > /backups/agent_platform_$(date +\%Y\%m\%d).sql.gz
```

## SSL/TLS Configuration

For production, always use SSL:

```bash
# Require SSL
DATABASE_URL=postgresql://user:pass@host:5432/db?sslmode=require

# Verify SSL certificate
DATABASE_URL=postgresql://user:pass@host:5432/db?sslmode=verify-full
```

## Performance Tuning

### PostgreSQL Configuration

Key PostgreSQL settings for optimal performance:

```sql
-- In postgresql.conf
shared_buffers = 256MB
work_mem = 16MB
maintenance_work_mem = 128MB
effective_cache_size = 1GB
```

### Index Monitoring

Check for unused indexes:
```sql
SELECT schemaname, tablename, indexname, idx_scan
FROM pg_stat_user_indexes
WHERE idx_scan = 0
ORDER BY pg_relation_size(indexrelid) DESC;
```

## Troubleshooting

### Connection Refused

```
Error: connect ECONNREFUSED
```

Verify:
1. PostgreSQL is running: `pg_isready`
2. Host/port are correct in `DATABASE_URL`
3. Firewall allows connections

### Authentication Failed

```
Error: password authentication failed
```

Verify:
1. Username and password in `DATABASE_URL`
2. User exists: `psql -U postgres -c "\du"`
3. `pg_hba.conf` allows password authentication

### Migration Errors

If a migration fails partway:
1. Check the `migrations` table for applied versions
2. Manually inspect partial schema
3. Fix the issue and retry or rollback manually

```sql
SELECT * FROM migrations ORDER BY version DESC LIMIT 5;
```

## Migration from SQLite

To migrate from SQLite to PostgreSQL:

1. Export data from SQLite:
   ```bash
   sqlite3 data/app.db .dump > sqlite_dump.sql
   ```

2. Convert SQL syntax (see schema differences above)

3. Create PostgreSQL database:
   ```bash
   createdb agent_platform
   ```

4. Apply PostgreSQL migrations:
   ```typescript
   // Use the migration runner
   ```

5. Import converted data

## High Availability

For production deployments:

1. Use connection pooling (PgBouncer)
2. Set up streaming replication
3. Configure automatic failover (Patroni)
4. Use a managed PostgreSQL service (AWS RDS, Google Cloud SQL, Azure Database)
