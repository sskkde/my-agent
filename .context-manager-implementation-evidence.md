## Pipeline Stages Implemented

### 1. Normalize
- Assigns default values for missing fields
- Infers semantic types from source types
- Estimates token counts for content
- Identifies pair groups for integrity protection

### 2. Filter
- Removes expired items (validUntil < now)
- Handles superseded items (keeps only newest per supersedesKey)

### 3. Dedup
- Removes duplicate items based on dedupeKey
- Preserves first occurrence

### 4. Score/Rank
- Composite scoring algorithm:
  - Priority: 40% weight
  - Recency: 30% weight
  - Relevance: 20% weight
  - Authority: 10% weight
- Sorts by final score (descending)
- Extracts pinned items separately

### 5. Budgeted Selection
- Respects global token budget
- Respects per-source token budgets
- Handles pair integrity:
  - If one item of a pair is selected, both are included
  - Pair token count checked as a unit
- Excludes low-priority items first when budget exceeded
- Generates compact hints when utilization > 80%

## Pair Integrity Protection

Pairs that must stay together (as per documentation lines 408-429):
- tool_use/result pairs
- Approval request/response pairs
- Artifacts with their references
- Workflow step IO
- BackgroundRun/SubagentRun pairs

Implementation:
- Items with matching pairId and requiresPairIntegrity=true are treated as units
- During selection, if any item from a pair is selected, all paired items are included
- Pair token counts are checked as a unit against budget

## Key Design Decisions

1. **No LLM calls** - Context decisions are rule-based, not LLM-based
2. **Store interfaces consumed, not implemented** - Uses existing store types
3. **Tool schema exposure not handled** - Tool Plane owns that responsibility
4. **Pair integrity at selection stage** - Ensures pairs are treated as atomic units
5. **Comprehensive scoring** - Multiple factors weighted for ranking
6. **Compact hints** - Early warning system for context window pressure
